Tablas ORDER y ORDERENTRY (esquema y relaciones)
Se adjunta archivo Excel con las relaciones
 1. Tabla: Order
Propsito: Representa una orden de venta completa (cabecera).
Llave primaria: ID
Campos principales:
StoreID (int)  ID de la tienda donde se genera la orden. 40 por defecto en la tienda virtual
ID (int, identity)  Identificador nico de la orden.
Time (datetime)  Fecha y hora de creacin de la orden.
Type (int)  Tipo de orden (venta, devolucin, cotizacin, etc.).
CustomerID (int)  Identificador del cliente.
Deposit, Tax, Total (money)  Valores financieros de la orden.
SalesRepID (int)  Identificador del vendedor asociado.
ShippingServiceID, ShippingTrackingNumber  Datos de envo.
Comment, ShippingNotes  Comentarios del usuario.
 2. Tabla: OrderEntry
Propsito: Representa los renglones (lneas) de productos dentro de una orden.
Llave primaria: ID
Relacin clave: OrderID  Order.ID
Campos principales:
OrderID (int)  ID de la orden padre.
ItemID (int)  ID del producto vendido.
Price, FullPrice, Cost (money)  Precio unitario, precio base y costo.
QuantityOnOrder, QuantityRTD (float)  Cantidades pedidas y despachadas.
SalesRepID, DiscountReasonCodeID, ReturnReasonCodeID  Atributos de operacin.
Description (nvarchar)  Descripcin del producto.
IsAddMoney (bit)  Indica si la lnea es un cargo adicional.
VoucherID  Si aplica, referencia a un cupn o voucher.
 3. Tabla: OrderHistory
Propsito: Auditora de movimientos de rdenes, especialmente depsitos y cobros.
Llave primaria: ID
Relacin clave: OrderID  Order.ID
Campos principales:
BatchNumber  Lote de procesamiento al que pertenece.
Date  Fecha del movimiento (auditora).
CashierID  Cajero responsable del movimiento.
DeltaDeposit (money)  Cambio en el depsito (positiva o negativa).
TransactionNumber  Nmero de transaccin dentro del batch.
Comment  Breve descripcin del movimiento.
